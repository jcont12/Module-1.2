Command Line SYNTAX

GIT

The command line is a text interface for the computer's operating system which allows us to move through our files just like the finder does. To access the command line, we use the terminal.

A filesystem organizes a computer's files and directories into a tree structure, always starting with the root directory. Each parent directory can contain more child directories and files. Remember directory=folder

Another helpful term to know while in the command line is "environment". The environment refers to the preferences and settings of the current user.The nano editor is a command line text editor used to configure the environment. The standard environment settings are stored in ~/.bash_profile, which you can edit with nano.
Finally, environment variables are variables that can be used across commands and programs and hold information about the environment.


If you want the complete list of command line commands from codeacademy, click here.

pwd - (print working directory) outputs the name of the current working directory.
ls - (list) lists all files and directories in the working directory.
cd - (change directory) switches you into the directory you specify. If you want to go back, you must type cd .. and if you need to go back further type cd ../..
mkdir - (make directory) creates a new directory in the working directory.
touch - creates a new file inside the working directory
cat - outputs content of a file into the directory
wc - (word count) outputs the number of lines, words and characters of a file
sort - sorts lines of text alphabetically.
uniq- filters duplicate, adjacent lines of text.
grep - (global regular expression print) searches for a text pattern and outputs it, but is case sensitive. If you dont want it to be case sensitive, add -i. if you add -R it will find searched text in files. If you add -Rl, it will find files that have said text
sed 's/"find"/"replace' - (stream editor) searches for a text pattern, modifies it, and outputs it. it must always begin with 's' substitute, followed by the text to find and followed by what to replace it with...it will only be replaced the first time it is found in a line, so if you want all instances to be replaced end it with /g for globally (ex: sed 's/dolomita/marmol/g' master)
clear - clears all written code in terminal and goes to space 1

There are also options that modify commands that modify the behaviour of commands, such as:

  • -a (ex:ls -a) : (all contents) lists all contents of a directory, including hidden files and directories
  • -l : (long format) lists all contents in long format
  • -t : (time format) orders files and directories by the time they were last modified
  • -alt : all options used together.
> - redirects standard output of a command to a file, overwriting previous content.
>> - redirects standard output of a command to a file, appending new content to old content.
< - redirects standard input to a command.
| - (is a pipe) redirects standard output of the command to the left and pipes it as standard input to the command to the right

It is also important to learn about nano which is the command line text editor. Think of nano as your "word" for the command line. In nano you can create a new file, edit the text inside it and save it.

nano "filename".txt - Creates a new file, and you can start adding text to it.
nano ~/.bash_profile - Opens the file used to store environment settings
source ~/.bash_profile - activates the changes in ~/.bash_profile for the current session. Instead of closing the terminal and needing to start a new session, source makes the changes available right away in the session we are in.

Once inside nano, use the following commands and keyboard shortcuts:

export VARIABLE="value" - sets and exports an environment variable... it helps for you to set shortcuts to variables
USER - The name of the current user
PS1 - the command prompt (standard $ is changed to another symbol)
HOME - Home directory, usually not customized (standard $)
PATH - returns a colon separated list of file paths. It is customized in advanced cases.
env - Returns a list of environment variables
alias- createa a keyboard shortcut for commands...for example: alias p= "pwd"
Ctrl + O - saves a file. 'O' stands for output. Remember to press enter to save
Ctrl + X - exits the file
Ctrl + G - Opens a help menu
